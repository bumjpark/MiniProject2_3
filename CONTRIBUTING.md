## Commit Type

| Type | Description |
|---|---|
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| refactor | 코드 리팩토링 |
| docs | 문서 수정 |
| test | 테스트 코드 추가 |
| chore | 설정 파일 및 환경 수정 |
| style | 코드 포맷 수정 |

---

## Commit Rule

- 하나의 커밋에는 하나의 작업만 포함한다.
- 커밋 메시지는 작업 내용을 명확하게 작성한다.
- 명령형 또는 간결한 문장으로 작성한다.
